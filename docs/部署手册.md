## 一、环境要求

| 软件 | 版本 |
|------|------|
| Docker Desktop | 4.x+（启用 BuildKit） |
| 内存 | 建议 ≥ 8GB（全栈构建较吃内存） |
| 磁盘 | 首次构建约 2～4GB 镜像/缓存 |

---

## 二、部署模式

| 命令 | 说明 |
|------|------|
| `docker compose up -d` | **仅 MySQL + Redis**（本地 `spring-boot:run` 开发） |
| `docker compose --profile app up -d` | **全栈（推荐）**：已有镜像时约 2 分钟 |
| `docker compose --profile app up -d --build` | 改代码后重建镜像（首次 15～30 分钟） |

> **注意：** `--profile app` 已包含 Nacos，无需再单独加 `--profile nacos`。

---

## 三、全栈一键部署

```powershell
cd mall-master

# 1. 环境变量（可选，默认即可演示）
copy .env.example .env

# 2. 减少 PowerShell 进度刷屏（可选）
$env:COMPOSE_PROGRESS = "plain"

# 3. 已有镜像时直接启动（推荐）
docker compose --profile app up -d

# 3b. 首次或改代码后重建
# $env:DOCKER_BUILDKIT = 1
# docker compose --profile app up -d --build

# 4. 查看状态
docker compose --profile app ps
```

**预期：** 下列 **11 个**容器均为 `Up` 且 `healthy`：

`relife-mysql`、`relife-mysql-slave`、`relife-redis`、`relife-nacos`、`relife-user`、`relife-item`、`relife-trade`、`relife-stock`、`relife-ai`、`relife-gateway`、`relife-nginx`

（`relife-nacos-config-init` 执行一次后应 exit 0，并可在配置列表看到 `campus-shared.yaml`。）

## 四、访问与验证

| 入口 | 地址 |
|------|------|
| 前端（Nginx） | http://localhost |
| API 网关（直连） | http://localhost:8080/api/...（`GET /` 无页面，404 正常） |
| Nacos 控制台 | http://localhost:8849/index.html（首次需初始化 `nacos` 密码） |
| Nacos Open API | http://localhost:8848 |

**快速验证：**

```powershell
curl.exe http://localhost/api/categories
curl.exe "http://localhost/api/items?page=1&size=5"
```

登录（PowerShell 推荐 `Invoke-RestMethod`，避免 JSON 转义）：

```powershell
$auth = Invoke-RestMethod -Uri http://localhost/api/auth/login -Method Post `
  -ContentType "application/json" `
  -Body (@{ loginName = "demo01"; password = "12345678" } | ConvertTo-Json)
$token = $auth.data.accessToken
Invoke-RestMethod -Uri http://localhost/api/cart -Headers @{ Authorization = "Bearer $token" }
```

## 六、环境变量（`.env`）

见 [.env.example](../.env.example)。关键项：

| 变量 | 说明 |
|------|------|
| `JWT_SECRET` | 各服务 JWT 密钥须一致 |
| `INTERNAL_TOKEN` | Feign 内部调用令牌 |
| `NACOS_DISCOVERY_ENABLED` | 全栈默认 `true` |
| `AI_LLM_ENABLED` | 默认 `false`，AI 走模板降级（报告演示推荐） |
| `OPENAI_API_KEY` | 设为有效 Key 且 `AI_LLM_ENABLED=true` 时走真实 LLM |
| `GATEWAY_RATE_LIMIT` | 限流阈值（默认 10 次/10 秒） |

## 七、构建说明

- **后端镜像：** `docker/Dockerfile.spring` 多阶段（JDK 21 编译 → JRE 21 运行）
- **前端镜像：** `docker/Dockerfile.web`（Node 20 构建 `dist` → Nginx 托管）
- **Nginx 配置：** `docker/nginx/default.conf`（`/` 静态资源，`/api/` 反代 Gateway）
- **数据库初始化：** `docker/mysql/init/` 挂载至 MySQL 首次启动脚本
- **Nacos 共享配置：** `docker/nacos/campus-shared.yaml`（限流、JWT 过期等）

## 八、常用运维命令

```powershell
# 停止全栈（保留数据卷）
docker compose --profile app down

# 停止并删除数据卷（清空数据库，慎用）
docker compose --profile app down -v

# 查看某服务日志
docker logs -f relife-gateway

# 仅重建单个服务（改代码后）
docker compose --profile app up -d --build item-service trade-service
```

## 九、故障排查

| 现象 | 处理 |
|------|------|
| `docker compose up` 无业务容器 | 未加 `--profile app` |
| 构建 Maven 超时 | 确保 `DOCKER_BUILDKIT=1`；重试 `docker compose --profile app build` |
| 3306 / 8080 / 80 端口占用 | 停本机 MySQL 或旧 `spring-boot:run`：`netstat -ano \| findstr :8080` |
| 服务 unhealthy | `docker logs relife-<服务名>`；MySQL 未就绪时等待 1～2 分钟 |
| `UnknownHostException: mysql` | 确认 mysql/redis 已加入 `relife-net` |
| Nacos exit 255 / unhealthy | 见问题日志 #18–#20（`NACOS_AUTH_TOKEN`、healthcheck） |
| Nacos 8848/8849 端口冲突 | 本机已有其他 Nacos（如 eleme 项目）占用端口 | 先 `docker stop <other-nacos>`，再启 `relife-nacos` |
| 发布物品 503，容器均 healthy | Feign 误连 `127.0.0.1:8104`；重建 `item-service`（问题日志 #24） |
| 购物车/通知 500 | ShardingSphere 缺 `cart_entry`/`trade_notify`；重建 `trade-service`（#25） |
| 前端 502 | Gateway 未 healthy，先 `docker logs relife-gateway` |
| 中文乱码（仅终端） | PowerShell 编码；浏览器/API 正常 |
| 主从未同步 | 首次 GTID：`docker compose --profile app down -v` 后重建 |
| 限流 429 | 正常；调 `GATEWAY_RATE_LIMIT` 或等待窗口过期 |

治理验收详见 [联调与验收清单.md](./联调与验收清单.md) §三。

## 十、与本地开发的关系

| 场景 | 做法 |
|------|------|
| 日常改代码 | `docker compose up -d` + 本地 `mvnw spring-boot:run` + `npm run dev` |
| 提交/报告验收 | `docker compose --profile app up -d`（已有镜像）；改代码后加 `--build` |
| 正式联调清单 | 见 [联调与验收清单.md](./联调与验收清单.md) |

本地开发时 `NACOS_DISCOVERY_ENABLED` 默认为 `false`，Feign 使用 `127.0.0.1:810x` 固定 URL；与 Docker 全栈模式不同，切换时注意停掉冲突进程。
