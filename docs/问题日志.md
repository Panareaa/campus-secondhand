## 详细记录

### P-001 Maven Wrapper 路径与命令（2026-06-30）

**现象：** 在 `final_project` 目录执行 `./mvnw` 提示找不到命令。

**原因：** Maven Wrapper 在 `mall-master/` 子目录；Windows PowerShell 应使用 `.\mvnw.cmd`。

**做法：**
```powershell
cd mall-master
.\mvnw.cmd clean install -DskipTests
```

### P-002 单模块启动依赖未安装（2026-06-30）

**现象：** `.\mvnw.cmd -pl campus-user-service spring-boot:run` 报 `campus-common-web` 找不到。

**原因：** `-pl` 只编译当前模块，兄弟模块 jar 未 install 到 `~/.m2`。

**做法：** 全量 install 后再启动单个服务。

---

### P-003 MyBatis-Plus 与 Spring Boot 4 不兼容（2026-06-30，关键）

**现象：**
```
Property 'sqlSessionFactory' or 'sqlSessionTemplate' are required
```
Gateway 能启动，User/Item 启动失败。

**原因：** Spring Boot **4.0.0** 与 `mybatis-plus-spring-boot3-starter` 不兼容，自动配置不生效。

**修复：**
- 父 POM：`mybatis-plus.version` → `3.5.15`，引入 `mybatis-plus-bom`
- 各业务服务：`mybatis-plus-spring-boot4-starter` + `spring-boot-starter-jdbc`

**验证：** 日志出现 MyBatis-Plus `3.5.15` 横幅，服务正常启动。

---

### P-004 端口占用与重复启动（2026-06-30 / 2026-07-01）

**现象：** 第二次 `spring-boot:run` 报 `Port 810x was already in use`。

**原因：** 每个服务只需一个进程；IDE/多终端重复启动。

**做法：**
```powershell
netstat -ano | findstr ":8101 :8102 :8103 :8104 :8080"
taskkill /PID <真实PID> /F
```

**注意：** 若冒烟测试已通过，说明服务已在跑，**无需再启动**。

---

### P-005 Docker Nacos 镜像拉取慢（2026-06-30）

**现象：** `docker compose up -d` 长时间卡在 `nacos Pulling`，或代理 `dockerproxy.com` 失效。

**原因：** Nacos 镜像体积大，国内网络/镜像源不稳定。

**做法：**
- 日常开发：仅 `docker compose up -d`（MySQL + Redis）
- 需要注册中心：`docker compose --profile nacos up -d`
- 配置 Docker Desktop 国内镜像加速（见 `README.md`）

### P-006 PowerShell 下 HTTP 测试 JSON 转义（2026-06-30）

**现象：**
- `curl.exe -d "{\"username\":...}"` → 500 + `JSON parse error`
- 末尾 `curl: (3) URL rejected: Bad hostname`

**原因：** PowerShell 对引号/反斜杠处理与 bash 不同，请求体被破坏。

**推荐写法：**
```powershell
Invoke-RestMethod -Uri http://localhost:8080/api/auth/register `
  -Method Post -ContentType "application/json" `
  -Body (@{campusId="2021001999";loginName="smoke99";password="12345678";nickname="smoke";contactInfo="smoke99@ynu.edu.cn"} | ConvertTo-Json)
```

### P-007 注册接口字段名错误（2026-06-30）

**现象：** 使用 `username`、`email` 注册失败。

**正确字段（§4.1）：** `campusId`、`loginName`、`password`（≥8 位）、`nickname`、`contactInfo`（可选）。

---

### P-008 业务错误码显示为 HTTP 500（2026-06-30）

**现象：** 重复学号注册时 `Invoke-RestMethod` 报「远程服务器返回错误: (500)」。

**原因：** `LOGIN_NAME_EXISTS(10003)` / `CAMPUS_ID_EXISTS(10004)` 在 `GlobalExceptionHandler` 中未映射到 HTTP 409。

**修复：** 完善 `resolveHttpStatus` 对 `1xxxx` 域错误码的映射 → 现返回 HTTP 409。

---

### P-011 Gateway `ObjectMapper` 缺失（2026-07-01，关键）

**现象：**
```
JwtAuthGlobalFilter required a bean of type 'com.fasterxml.jackson.databind.ObjectMapper' that could not be found
```

**原因：** Spring Boot 4 迁移至 Jackson 3，不再自动提供 `com.fasterxml.jackson.databind.ObjectMapper` bean；Gateway 过滤器仍依赖旧类型。

**修复：** 移除 `ObjectMapper` 注入，`writeError` 方法手写 JSON 字符串（与 `InternalAuthFilter` 一致）。

**验证：** Gateway 正常启动，`Netty started on port 8080`。

---

### P-012 购物车加购路径错误（2026-07-01）

**现象：** `POST http://localhost:8080/api/cart` → trade-service 报 `Request method 'POST' is not supported`，HTTP 500。

**原因：** 加购 API 为 `POST /api/cart/items`（`CartController`），`/api/cart` 仅支持 GET（列表）与 DELETE（清空）。

**做法：** 冒烟脚本使用：
```powershell
Invoke-RestMethod -Uri http://localhost:8080/api/cart/items -Method Post -Headers $bh `
  -ContentType "application/json" -Body (@{ itemId=2; quantity=1 } | ConvertTo-Json)
```

---

### P-013 重复注册返回 409（2026-07-01）

**现象：** 再次注册 `smoke100` 时 `Invoke-RestMethod` 报 `(409) 冲突`。

**原因：** 账号已在先前冒烟中创建；`10003`/`10004` 已正确映射为 HTTP 409。

**做法：** 跳过注册，直接登录：
```powershell
$buyer = Invoke-RestMethod -Uri http://localhost:8080/api/auth/login -Method Post `
  -ContentType "application/json" -Body (@{loginName="smoke100";password="12345678"} | ConvertTo-Json)
```

### P-021 ShardingSphere-JDBC 启动失败（2026-07-01）

**现象：** `campus-trade-service` 启动报错：

### P-023 Nacos 配置中心自动发布失败（2026-07-01，2026-07-03 已修复）

**现象：** `relife-nacos-config-init` 容器 `curl: (22) error 500/403`；浏览器访问 `http://localhost:8848/nacos` 无法进入 Nacos 3.x 控制台。

**原因：**
1. **Nacos 3.x 控制台** 独立在容器 **8080**，入口为 `/index.html`，不再是 2.x 的 `8848/nacos`。
2. 控制台默认开启登录；Admin API 发布配置需 **serverIdentity** 请求头。
3. v3 Admin API 的 `content` 必须是 **字符串表单字段**，不能用 multipart 文件上传（v1 的 `-F content=@file` 会 500）。

**修复：**
- `docker-compose.yml`：映射 `8849:8080`（避免与 Gateway `:8080` 冲突）；`NACOS_AUTH_CONSOLE_ENABLE=false`、`NACOS_AUTH_ADMIN_ENABLE=false`。
- `docker/nacos/publish-config.sh`：v3 API + `--data-urlencode "content@${CONFIG_FILE}"` + identity 头 + 发布校验。

**访问：**
- 控制台：**http://localhost:8849/index.html**（无需登录）
- Open API：http://localhost:8848

**验证：**
```powershell
docker compose --profile app run --rm nacos-config-init
curl "http://localhost:8848/nacos/v3/client/cs/config?dataId=campus-shared.yaml&groupName=DEFAULT_GROUP"
```

**注意：** 若本机已有其他 Nacos 容器占用 8848（如 `eleme-nacos`），需先停止后再启 `relife-nacos`。

### P-024 Docker 下发布物品 503（Feign 连 localhost）（2026-07-01）

**现象：** 浏览器发布物品时 `POST http://localhost/api/items` 返回 **503**；控制台同时可见 `/api/cart`、`/api/notifications` 500。`docker compose --profile app ps` 显示全部容器 **healthy**，易误判为「服务没开」。

**Gateway 日志：**
```
POST /api/items -> 503
```

**Item 服务日志（关键）：**
```
INSERT INTO idle_item ... Updates: 1
[StockFeignClient#initStock] ---> POST http://127.0.0.1:8104/internal/stock/init
Connection refused executing POST http://127.0.0.1:8104/internal/stock/init
```

**原因：** 物品写入 MySQL 成功，但初始化库存时 Feign 使用 `application.yml` 默认地址 `http://127.0.0.1:8104`。在 Docker 网络中，`127.0.0.1` 指向 **item 容器自身**，无法访问 `stock-service` 容器。虽已启用 `NACOS_DISCOVERY_ENABLED=true` 且 `@FeignClient(name = "campus-stock-service")`，但 `spring.cloud.openfeign.client.config.campus-stock-service.url` 被硬编码 URL 覆盖，导致不走服务发现。

**修复：**
- `campus-item-service/src/main/resources/application.yml`
- `campus-trade-service/src/main/resources/application.yml`

将 Feign 默认 URL 改为空（留空时走 Nacos）：
```yaml
campus:
  feign:
    stock-service-url: ${STOCK_SERVICE_URL:}
    item-service-url: ${ITEM_SERVICE_URL:}
    user-service-url: ${USER_SERVICE_URL:}
```

本地无 Nacos 时可在环境变量或 IDE 运行配置中设置，例如 `STOCK_SERVICE_URL=http://127.0.0.1:8104`。

**重建：**
```powershell
docker compose --profile app up -d --build item-service trade-service
```

**验证：** Item 日志出现 `POST http://campus-stock-service/internal/stock/init` → HTTP 200；`POST /api/items` 返回 code=0。

**注意：** 修复前多次重试可能已插入部分 `idle_item` 记录（事务因 Feign 异常回滚，但需留意列表中的重复草稿，可在「我的发布」手动删除）。

### P-025 ShardingSphere 未注册购物车/通知表（2026-07-01）

**现象：** 登录后访问购物车、通知页，`GET /api/cart`、`GET /api/notifications` 返回 **500**。

**Trade 服务日志：**
```
TableNotFoundException: Table or view 'trade_notify' does not exist.
```

（`cart_entry` 同理，访问购物车时触发。）

**原因：** 接入 ShardingSphere-JDBC 后，`sharding.yaml` 仅声明分片表 `trade_order`、`trade_line`。`cart_entry`、`trade_notify` 为普通单表，未在规则中注册时，ShardingSphere 元数据绑定失败，MyBatis 查询报「表不存在」。

**修复：** `campus-trade-service/src/main/resources/sharding.yaml` 增加 `!SINGLE` 规则：
```yaml
  - !SINGLE
    tables:
      - "ds_0.cart_entry"
      - "ds_0.trade_notify"
```

**验证：**
```powershell
docker compose --profile app up -d --build trade-service
# GET /api/cart、GET /api/notifications → code=0
```
